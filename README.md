# demo-book-catalog

Демо-проект: каталог книг на Yii2 + MySQL.

Гость просматривает книги и авторов, может оформить подписку на нового автора. Авторизованный пользователь управляет каталогом (CRUD книг и авторов). Отдельная страница — отчёт «топ-10 авторов по числу книг за указанный год».

В качестве дополнительной возможности — SMS-уведомление подписчикам при появлении новой книги автора (через `smspilot.ru` в режиме эмулятора).

## Стек

- PHP 8.4
- Yii2 (basic template)
- MySQL 8.4
- Docker + Docker Compose

## Что реализовано (по ТЗ)

- ✅ Сущность **Книга**: название, год, описание, ISBN, обложка (локально в `web/uploads/covers`).
- ✅ Сущность **Автор**: ФИО (фамилия / имя / отчество).
- ✅ Связь **M:N** — у книги может быть несколько авторов (`book_authors`).
- ✅ **Гость** — просмотр каталога и подписка на нового автора (на карточке `/author/view`).
- ✅ **Авторизованный пользователь** — CRUD книг и авторов (`AccessControl`-фильтр).
- ✅ **Отчёт «Топ-10 авторов» за год** — `/report/top-authors?year=YYYY`, доступен всем.
- ✅ **CRUD «Книга»** — просмотр всем, edit/delete только авторизованному.
- ✅ **CRUD «Автор»** — просмотр всем, edit/delete только авторизованному.
- ✅ **SMS-уведомление подписчикам** при появлении новой книги (smspilot.ru, тестовый ключ-эмулятор).
- ✅ **Миграции** вместо дампа БД (5 файлов в `migrations/`), консольный сидер `./bin/yii seed`.

## Запуск

Нужны Docker и Docker Compose v2. Одной командой:

```bash
./bin/setup
```

Скрипт поднимет контейнеры (`app`, `nginx`, `mysql`), поставит зависимости через `composer install`, накатит миграции и зальёт тестовые данные (без обложек — быстро). Идемпотентен: можно запускать повторно.

Опции:

```bash
./bin/setup --with-covers   # тестовые данные с обложками с picsum.photos (медленно)
./bin/setup --no-seed       # без сидинга, только миграции
./bin/setup --help          # справка
```

Если нужно вручную, по шагам:

```bash
docker compose up -d --remove-orphans
docker compose exec app composer install
docker compose exec app ./bin/yii migrate --interactive=0
docker compose exec app ./bin/yii seed --skipCovers=1
```

Откройте http://localhost:8000.

### SMS-уведомления

При создании новой книги подписчикам её авторов уходит SMS через [smspilot.ru](https://smspilot.ru). По умолчанию используется тестовый ключ-«эмулятор» — API принимает запрос, но реально не отправляет.

Для реальных отправок задайте переменные окружения перед `docker compose up`:

| Переменная             | Назначение                              |
|------------------------|-----------------------------------------|
| `SMS_PILOT_API_KEY`    | Ключ API smspilot.ru                    |
| `SMS_PILOT_FROM`       | Alpha-sender (по умолчанию `INFORM`)    |

Чтобы убедиться, что уведомления отрабатывают (особенно на эмуляторе, где реальное SMS никуда не приходит), смотрите в реальном времени:

```bash
tail -f runtime/logs/app.log | grep -E "SmsPilotClient|NewBookNotifier"
```

После создания книги в логе появятся строки вида:

```
[info][app\services\SmsPilotClient::send] SMS принято smspilot (+7912****7890): Новая книга «...» — ...
[info][app\services\NewBookNotifier::notify] Новая книга «...» (id=42): уведомлено 3 из 3 подписчиков
```

Сбои отправки логируются с уровнем `warning` в том же файле и содержат причину:

- **Нет связи с API** (`endpoint` недоступен, тайм-аут, DNS-сбой) — `SMS не отправлено (+...): нет связи с https://smspilot.ru/api.php`.
- **Битый JSON в ответе** (API вернул HTML-заглушку или мусор) — `некорректный JSON в ответе: ...`.
- **API вернул ошибку** (неверный ключ, антиспам, неправильный формат номера и т.п.) — `smspilot вернул ошибку <код> — <описание>`.

> На стандартном тестовом ключе-эмуляторе smspilot.ru сейчас возвращает ошибку **`226 — Antispam (sender not configured, alt. channels disabled)`**: alpha-sender `INFORM` (наш дефолт) не зарегистрирован под этим ключом. Это нормально для тестовой среды: запрос ушёл, API его обработал, антиспам-фильтр отверг — для проверки интеграции достаточно того, что в логе видна цепочка «принят → отвергнут API». Для прода нужно либо зарегистрировать sender в кабинете smspilot.ru, либо переопределить `SMS_PILOT_FROM` под зарегистрированный.

### Архитектура

**Слои.** Запрос проходит цепочку `controller → service → ActiveRecord → DB`, без перепрыгивания: контроллер не зовёт `$model->save()` напрямую, а делегирует сервису. Сервисы — единая точка входа для побочных эффектов (рассылка SMS, в перспективе welcome-сообщение, аудит).

**Контроллеры** (`controllers/`). Тонкие, типичный экшен — три шага: подгрузить ввод (`Model::load`), позвать сервис, отдать `render`/`redirect`. CSRF-валидация и фильтр методов (`VerbFilter`) — на уровне `behaviors()`. Доступ на запись — через `AccessControl` (`'roles' => ['@']`).

**Модели** (`models/`).
- `Book`, `Author`, `Subscription`, `BookAuthor` — ActiveRecord. Yii AR совмещает модель и репозиторий, поэтому отдельного слоя репозиториев нет (это идиоматично для Yii2 и не плодит лишний boilerplate).
- `BookSearch`, `AuthorSearch` — search-модели для GridView с фильтрами.
- `LoginForm`, `ContactForm` — form-модели на `yii\base\Model` (без таблицы).
- Виртуальные атрибуты в Book: `authorIds` (мульти-выбор авторов M:N) и `coverFile` (загружаемый файл).
- Оптимистичная блокировка через колонку `version` (см. миграцию `m260513_223352`): два пользователя не затрут изменения друг друга молча.

**Сервисы** (`services/`).
- `BookService` — `create()` + `update()`, инкапсулирует save и пост-эффекты (вызов `NewBookNotifier`).
- `SubscriptionService` — `subscribe()`. Сейчас тонкая обёртка над save, но точка входа уже выделена для будущих расширений.
- `NewBookNotifier` — собирает уникальные телефоны подписчиков (`DISTINCT JOIN` за один запрос), формирует SMS, дёргает клиента.
- `SmsPilotClient` — обёртка над HTTP-API smspilot.ru: формирует URL, парсит JSON, логирует исход (info при успехе, warning при сбоях, телефон маскируется в логе).
- Все сервисы автоматически резолвятся через DI Yii (типизированные параметры конструкторов). Регистрация только для `SmsPilotClient` (нужны параметры из `params.php`).

**Миграции** (`migrations/`). Пять файлов: создание `authors`, `books`, M:N-связки `book_authors`, таблицы `subscriptions`, добавление `version` для оптимистичной блокировки. Каждая колонка снабжена `->comment(...)` — схема самодокументирована.

**Сидер** (`commands/SeedController`). `./bin/yii seed` идемпотентен: чистит подписки/книги/авторов и заливает заново. Опции `--authorsCount`, `--booksCount`, `--skipCovers`. Faker (`ru_RU`) генерирует ФИО, ISBN, тексты; обложки — с picsum.photos.

**M:N-связь.** У книги может быть несколько авторов и наоборот. В форме книги — `Select2` с мульти-выбором; синхронизация `book_authors` идёт в `Book::afterSave()` под транзакцией (см. `transactions()`), чтобы откатить и саму книгу, если связи не лягут.

**Маршрутизация.** Топ-страницы (`/`, `/login`, `/about`, `/contact`) имеют короткие алиасы в `urlManager.rules`. Старые `/site/<action>` 301-редиректятся на канонический URL в `SiteController::beforeAction()` — для SEO и единообразия.
