# Тестовое задание

Получено от АНО ДПО «НАМО им. Н.А. Бородина», 2026-05-13. Лимит — не более 8 часов.

## Условие

Требуется разработать небольшой каталог книг, используя **Yii2** и базу данных **MySQL**. У одной книги может быть несколько авторов. Вёрстка не обязательна — достаточно рабочей функциональности.

### Сущности

1. **Книга** — содержит поля: название, год издания, краткое описание, ISBN, изображение обложки.
2. **Автор** — ФИО.

### Роли и их возможности

1. **Неавторизованный посетитель** — может просматривать каталог и оформлять подписку на появление новых книг конкретного автора.
2. **Авторизованный пользователь** — просмотр, создание, редактирование и удаление книг.

### Необходимые страницы

1. **Отчёт:** список 10 авторов, выпустивших наибольшее количество книг за указанный год. Доступен всем пользователям.
2. **CRUD для сущности «Книга»**. Доступен просмотр книг всем пользователям. Редактирование доступно только авторизованному пользователю.
3. **CRUD для сущности «Автор»**. Доступен просмотр всем пользователям. Редактирование доступно только авторизованному пользователю.

### Дополнительное задание (необязательно, но будет плюсом)

При появлении новых книг отправлять подписчикам (гостям) SMS-уведомление. Можно использовать сервис `smspilot.ru`, для теста — ключ «эмулятор» (сообщения реально отправляться не будут).

**Ограничение по времени:** не более 8 часов.
